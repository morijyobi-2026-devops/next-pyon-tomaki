提案システムの基本構成

管理者登録 → 学生選択 → 個人カレンダー → 次の授業表示

システムアーキテクチャ

📋 管理者

授業マスタ登録

  • 授業名
  • 曜日・時限
  • 教室
  • 担当教員

👤 学生

初回設定：履修授業を選択

  • ログイン
  • 授業一覧から選択
  • 個人カレンダー生成

以降の動作：

ホーム画面で現在時刻を確認 → 授業終了5分前から次の授業を優先表示 → 「次の教室：○○」を大きく表示

✅ この構成の利点：

  • 学校 Web システムのスクレイプ不要
  • UI/UX を自由に設計できる
  • 複数キャンパス・複数ユーザー対応可能
  • 保守性が高い
